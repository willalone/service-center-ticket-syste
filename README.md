# Service Center Ticket System

Система учёта заявок на ремонт оргтехники с графическим интерфейсом на PyQt5 и реляционной базой данных MySQL/MariaDB.

## Описание

Программный модуль для автоматизации процесса приёма и обработки заявок на ремонт оргтехники в сервисном центре. Поддерживает три роли пользователей: **клиент**, **оператор** и **мастер**.

## Функциональность

### Для клиента

- Создание и редактирование заявок на ремонт
- Просмотр информации о регистрации заявки
- Просмотр истории взаимодействия и переписки
- Прикрепление файлов к заявкам
- Получение оповещений о выполнении заявки

### Для оператора

- Просмотр списка всех заявок
- Распределение заявок по группам операторов, ответственным и наблюдателям
- Определение статусов, приоритетов и типов заявок
- Назначение мастеров на заявки
- Поиск в электронном архиве
- Поиск и удаление дубликатов заявок

### Для мастера

- Просмотр назначенных заявок, отсортированных по приоритету
- Отправка заявки на заказ запчастей
- Изменение статуса заявки
- Формирование отчёта о выполненной работе
- Прикрепление файлов (фото) к заявкам
- Просмотр истории взаимодействия и переписки

## Технологии

- **Python 3.7+**
- **PyQt5** — графический интерфейс
- **MySQL/MariaDB** — база данных
- **mysql-connector-python** — драйвер для работы с БД

## Установка

1. Клонируйте репозиторий:

```bash
git clone https://github.com/willalone/service-center-ticket-syste.git
cd service-center-ticket-syste
```

2. Установите зависимости:

```bash
pip install PyQt5 mysql-connector-python
```

3. Настройте базу данных:

   - Убедитесь, что у вас установлен MySQL или MariaDB
   - Выполните SQL-скрипт для создания БД:

   ```bash
   mysql -u root -p < service_center.sql
   ```

4. Настройте подключение к БД:
   - Откройте файл `db_connection.py`
   - Измените параметры в `DB_CONFIG`:
   ```python
   DB_CONFIG = {
       "host": "localhost",
       "user": "ваш_пользователь",
       "password": "ваш_пароль",
       "database": "service_center",
       "auth_plugin": "mysql_native_password",
   }
   ```

## Запуск

```bash
python3 ticket_gui.py
```

Откроется окно приложения с тремя вкладками: **Клиент**, **Оператор**, **Мастер**.

## Структура проекта

- `ticket_gui.py` — графический интерфейс (PyQt5)
- `db_connection.py` — модуль для работы с базой данных
- `ticket_system.py` — модели данных (используется как вспомогательный модуль)
- `service_center.sql` — SQL-скрипт для создания структуры БД

## База данных

База данных `service_center` содержит следующие таблицы:

- `roles` — роли пользователей
- `users` — пользователи системы
- `equipment_types` — типы оргтехники
- `equipment` — модели оборудования
- `request_statuses` — статусы заявок
- `requests` — заявки на ремонт
- `parts` — запчасти
- `request_parts` — связь заявок и запчастей
- `comments` — история взаимодействий (комментарии)
- `attachments` — прикреплённые файлы
